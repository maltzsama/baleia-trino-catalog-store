### N5 — `${baleia-secret:...}` placeholder syntax is safe in `CREATE CATALOG`

Verified in 482 source. The DDL path is:

```
CreateCatalogTask.execute
    → MetadataManager.createCatalog       (MetadataManager.java:849)
        → CatalogManager.createCatalog    (CatalogManager.java:55)
            → CoordinatorDynamicCatalogManager.createCatalog   (CoordinatorDynamicCatalogManager.java:257)
                → catalogStore.createCatalogProperties(...)    (line 273)
                → catalogStore.addOrReplaceCatalog(...)        (line 280, after the in-memory register)
```

`CreateCatalogTask` evaluates each property with `evaluateProperty(...)` (a SQL
expression evaluator) and passes the resulting string map straight through to
`MetadataManager.createCatalog`. **No `secretsResolver.getResolvedConfiguration`**
appears on this path. So our placeholder travels in verbatim — only our own
`SecretResolver` interprets it in `BaleiaCatalogStore.createCatalogProperties`.

→ On Trino upgrade, re-grep `CreateCatalogTask`, `MetadataManager.createCatalog`, and
`CoordinatorDynamicCatalogManager.createCatalog` for any new `secretsResolver` /
`getResolvedConfiguration` reference. If one shows up, switch `SecretResolver`'s
placeholder to a syntax outside Airlift's `${provider:key}` shape (e.g.
`@baleia-secret[catalog:key]`) so it doesn't get mangled. Adjust the Go
`CREATE CATALOG` generator at the same time.

## `FileCatalogStore` (reference: the built-in implementation we copy the skeleton from)

- `computeCatalogVersion` is **SHA-256** with:
  ```java
  hasher.putUnencodedChars("catalog-hash");          // fixed prefix
  hashLengthPrefixedString(catalogName.toString());
  hashLengthPrefixedString(connectorName.toString());
  hasher.putInt(properties.size());
  ImmutableSortedMap.copyOf(properties).forEach((k, v) -> {
      hashLengthPrefixedString(k);
      hashLengthPrefixedString(v);
  });
  return new CatalogVersion(hasher.hash().toString());

  private static void hashLengthPrefixedString(Hasher hasher, String value) {
      hasher.putInt(value.length());            // little-endian (Guava)
      hasher.putUnencodedChars(value);          // UTF-16LE (Guava)
  }
  ```
- This is the algorithm the Go implementation must reproduce **byte-for-byte**. Warning:
  little-endian + UTF-16LE confirmed (inherited from Guava `Hasher`).
- `createCatalogProperties(catalogName, connectorName, properties)` builds:
  ```java
  new CatalogProperties(catalogName, computeCatalogVersion(catalogName, connectorName, properties), connectorName, ImmutableMap.copyOf(properties))
  ```
  → No `CatalogHandle` involved. The version is the truth.
- In `addOrReplaceCatalog`: `catalogProperties.name()`, `connectorName()`, `version()`, `properties()`.
- In loadProperties: removes `connector.name` from the loaded properties (here the DB already has it separate).
- On persistence error, throws `TrinoException(CATALOG_STORE_ERROR, "Could not store catalog properties")`.

## Operational summary for the plugin

- Implement `CatalogStore` + `CatalogStoreFactory` + `Plugin.getCatalogStoreFactories()`.
- `createCatalogProperties(CatalogName, ConnectorName, Map)` → resolves secrets, computes SHA-256
  version, returns `new CatalogProperties(name, version, connector, ImmutableMap.copyOf(resolved))`.
- `addOrReplaceCatalog(CatalogProperties)` → UPSERTs the row with `version().toString()`.
- `removeCatalog(CatalogName)` → soft delete.
- `getCatalogs()` → SELECT enabled rows; `StoredCatalog.loadProperties()` resolves secrets and
  returns `CatalogProperties` consistent on `name()`.
- On persistence error: `throw new TrinoException(CATALOG_STORE_ERROR, "...")` (do not swallow).
